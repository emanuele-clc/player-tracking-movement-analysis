Original contribution: off-ball space creation score

Honest caveat, shown rather than hidden: this metric is validated as mathematically correct (a synthetic 4-players-in-4-quadrants self-test gives 0.00% error against the exact expected area split - run python src/space_creation.py --self-test yourself). But the scores below, computed on the current real test clip, are numerically unstable - some values exceed 10,000 m² per meter of displacement. That's an honest symptom of too little data (only 13 of 114 frames have both teams visible at once in this box-focused clip), not a bug in the method. A meaningful, stable player ranking needs a full match's worth of tracking data.

Pipeline status

  • 1. Data access (SoccerNet) - request pending, script ready
  • 2. Detection & tracking (YOLOv8 + ByteTrack) - done, validated on real video
  • 3. Pitch calibration (automatic line-detection homography) - done, validated to ~8cm mean error
  • 4. Team/role assignment (jersey-color clustering) - done, validated on real footage
  • 5. Tracking dataset assembly (+ speed) - done, validated
  • 6. Movement analysis (role clustering + heatmaps) - done, validated
  • 7. Original contribution (space-creation score) - method validated, real ranking needs more data
  • 8. Dashboard (this page) - done

Every stage above runs on real video with real output, not placeholder data - but several honest limitations (ball-tracking fragmentation with generic COCO weights, sparse teammate co-visibility in a short single-angle clip) are called out explicitly in this dashboard and in the repo's README rather than hidden, matching this project's companion xG model's approach to reporting results.

Built end-to-end: real detection (YOLOv8/ByteTrack) → automatic pitch-line homography calibration → jersey-color team clustering → role clustering & heatmaps → Voronoi-based space-creation metric. See the repository README for full methodology, honest limitations, and reproduction steps.
